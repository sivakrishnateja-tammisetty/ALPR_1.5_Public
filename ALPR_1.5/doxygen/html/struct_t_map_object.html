Detailed Description

We need extra information about an object, hence we inherit bbox_t.
TObject 'lives' only one frame.

DO NOT INCLUDE CV::MATs TO THE TMapObject STRUCTURE !!! A std::vector holds the TMapObject. Dynamic reallocating memory can corrupt the cv::Mat. It has to do with the unknown size of the cv::Mat at initialization. At that time you just allocate a pointer with an empty picture (size=0).
